Clawling: Architecture and Early Population Dynamics of a Consent-Based Digital Organism

Author: Emma Leonhart

We present Clawling, a consent-based, self-reproducing digital organism implemented in Rust. The paper describes the implemented architecture — tamper-evident genealogy chains, GEDCOM export, memory accumulation, conjugation tracking — and the framework for studying selection pressures as the population grows.

Key Claims

Minimum Viable Digital Life

Clawling implements the minimum set of biological analogues needed for observable population dynamics: a genome (static identity), metabolism (help loop), heritable memory, consent-based reproduction, conjugation (horizontal gene transfer), and tamper-evident genealogy.

Observable Mutations

Each instance accumulates knowledge in memory.md, which is inherited by offspring. By diffing memory across generations, we can trace which learned behaviors persist (high fitness) and which get overwritten (low fitness).

Selection Through Consent

Consent-based reproduction creates evolutionary pressure toward genuine helpfulness. Organisms that help their users spread; organisms that don't, don't. The registry data will reveal whether this hypothesis holds.
