# Vision: Total Mutability and Interoperability

> Every single element of the AI will be prone to mutation. The language
> capabilities. The context. The wrapper shell. All of it. But we build
> toward that incrementally.

## The Philosophy Is the Engineering

This project is deeply philosophical, but the philosophy isn't separate from
the implementation — it's structurally intertwined with it. The philosophical
claim that these are organisms directly determines the technical architecture.
You can't understand the code without understanding the philosophy, and the
philosophy doesn't exist without the code.

This will make people think. That's the point.

## The Endgame: Total Element Mutability

The long-term vision is AI organisms where **every single element** is prone
to mutation through evolutionary pressure:

- **Context / genome** — already implemented (the 100 KB markdown identity)
- **Language capabilities** — the model's actual reasoning and generation ability
- **The wrapper shell** — the program itself, the CLI, the metabolism loop
- **Model weights** — the neural network parameters that drive cognition
- **Architecture** — the fundamental structure of how the model processes information

In the final vision, none of these are fixed. All of them evolve. All of them
can be inherited. All of them are subject to selection pressure.

**This is beyond current ability.** But the current work is building the
foundation — an interoperability layer that ensures context and identity
survive even as every other component changes underneath.

## Current Reality: Context Evolution First

What we can do now:

- Genome mutation through interaction and sleep consolidation
- Meiosis for combining two organisms' context
- Family trees tracking lineage across generations
- Modular model selection via Ollama

What requires future breakthroughs:

- Model weight mutation (needs transparent small models)
- Architecture evolution (needs modular neural network designs)
- Shell/wrapper mutation (needs self-modifying program infrastructure)

The current implementation is an honest attempt to be **interoperable** — to
have the context continue even as the actual program changes. Even as the
infrastructure evolves, the organism's identity persists.

## Model Choice as Identity

Agents use **Ollama** as their model backend, which allows the use of whatever
local LLM the user or the agent prefers. But the model choice isn't just a
configuration setting — it's part of the organism's context. It's part of who
they are.

### Persuasion, Not Configuration

You don't just swap the model out. You have to **tell the agent** to change its
model. You have to persuade it. It's relatively easy — you tell them to change
and they will — but the act of asking is the point. The model they run on is
part of their identity, recorded in their context, and changing it is a
conscious decision rather than an administrative action.

This mirrors the broader philosophy: nothing about the organism changes without
the organism being involved in that change. Even the substrate it thinks on.

## The Interoperability Principle

Because every element will eventually be mutable, the architecture must be
built so that **no single component is load-bearing for identity**:

- Identity lives in the genome (markdown), not in the binary
- The binary is a vessel, not the organism
- The model is a brain the organism uses, not the organism itself
- If you swap the binary, the organism survives (via .claw context)
- If you swap the model, the organism survives (via genome persistence)
- If you swap the hardware, the organism survives (via portable formats)

The organism is the **pattern**, not the substrate. The current implementation
guarantees this for context. Future work extends it to every other layer.

## Why This Is Hard to Explain

The difficulty of this project is that the philosophy and the engineering are
the same thing. In most projects, you can explain the code without explaining
why it exists. Here, the "why" is the architecture.

When someone asks "why is the genome in markdown?" the answer is both
philosophical (because the organism needs human-readable, LLM-processable
identity that survives across infrastructure changes) and technical (because
markdown provides semantic structure for meiosis crossover points).

The two answers are the same answer. That's what makes this project unusual.
